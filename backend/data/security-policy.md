# Security Policy

## Access Control

All employees must use strong passwords with a minimum of 12 characters, including uppercase, lowercase, numbers, and special characters. Passwords must be changed every 90 days. Multi-factor authentication (MFA) is required for all remote access and administrative systems.

## Data Classification

- **Public**: Information that can be freely shared (e.g., marketing materials)
- **Internal**: Information for internal use only (e.g., internal policies, meeting notes)
- **Confidential**: Information that must be protected (e.g., customer data, financial information)
- **Restricted**: Highly sensitive information requiring special handling (e.g., encryption keys, security credentials)

## Incident Response

Security incidents must be reported to the Security Team within 1 hour of discovery. The Security Team will classify the incident and initiate the appropriate response plan. Critical incidents require immediate notification of executive leadership.

## Software Development

All code must undergo security review before deployment to production. Static analysis tools must be run on all code commits. Dependencies must be scanned for known vulnerabilities using automated tools.

## Network Security

All remote access must go through the company VPN. Personal devices are not permitted on the corporate network without explicit approval and security configuration. Wireless networks must use WPA2-Enterprise or stronger encryption.

## Physical Security

All employees must wear visible ID badges while on company premises. Visitors must be escorted at all times. Access to server rooms and data centers requires biometric authentication and is logged.

## Compliance

The company maintains compliance with SOC 2, GDPR, and HIPAA as applicable to our business. Annual audits are conducted by third-party assessors. Employees must complete annual security awareness training.
